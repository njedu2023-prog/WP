# WP 系统说明

WP 用于筛选今日涨幅超过 8%、前一日未涨停、今日未涨停的 A 股股票，并按尾盘次日收益条件排序输出 Top50。基础特征模型为 `wp_rule_v2_1`，尾盘排序模型为 `tail_profit_v1`。

当前主流程：

1. 根据北京时间和 A 股交易时段解析当前应完成的数据时间槽。
2. WP 直连 Tushare 获取实时行情、日线、每日指标、涨停状态等源数据，并拉取 `a-share-top3-data` 当前处理器代码。
3. 复用上游处理器完成与原链路一致的字段归一化、历史特征、板块特征、前一日涨停和数据质量加工，生成 `data/direct/latest/wp_latest_rank_input.csv`。
4. 对直连结果执行关键字段、交易日期、历史覆盖率和候选数量检查。直连失败时才读取上游 latest CSV/缓存，并在健康状态和报告中明确标记回退。
5. 硬过滤：今日涨幅超过 8%、前一日未涨停、今日未涨停、成交额达到配置门槛，并排除 ST、停牌/无成交、退市整理、新股上市未满配置天数、严重数据缺失票。
6. 计算强板块、强个股、资金承接、动量、资金、形态、风险惩罚；字段包括成交额放大、日内收盘位置、尾盘/VWAP、3/5/10/20 日动量、均线位置、20 日突破、龙虎榜、热门题材等。
7. 保留 `wp_score`、`ranking_score` 和 `p_limitup_t1` 作为解释字段。
8. 在全部候选中计算 `tail_profit_score`，硬限制涨幅不超过 12%、风险分不超过 45、收盘位置不低于 50、5 日成交额放大不超过 2.5 倍。
9. 按 `tail_profit_v1` 统一排序后截取 Top50，14:35 后最多生成 1 支主观察票；无合格票时空仓。
10. 原子提交 CSV、JSON、HTML 并发布 GitHub Pages；REST Git Data API 暂时失败时使用 GraphQL 原子提交回退。
11. 支持手动历史区间回测，输出 Top10/20/50、最终买入观察名单、AUC、Brier、次日涨幅和回撤指标。

尾盘观察计划用于 14:35 手动运行后的人工决策流程。用户观察到 14:50，只有涨幅仍大于 8%、未涨停、尾盘没有急回落且板块未转弱时才考虑执行。该模块不做自动下单。

历史日线代理回测只用于工程回归，不作为 14:35 实盘收益证明。累计验证只统计 `tail_profit_v1` 的最终尾盘名单，避免与旧模型记录混算。

实时模式会检查行情日期和计划时间槽。交易时间内若直连数据没有覆盖应完成时间槽，监控任务会触发一次受并发锁保护的强制修复，避免把旧行情误当成当前数据。

自动任务会在 A 股交易时间窗口内运行，并通过 Tushare 交易日历判断是否开市。GitHub cron 属于尽力调度，系统用 5 分钟心跳、时间槽去重、监控自愈和并发锁降低延迟与漏跑风险，但外部平台不可用时不能承诺 100% 准点。
