# WP 系统说明

WP 用于筛选今日涨幅超过 8%、前一日未涨停、今日未涨停的 A 股股票，并按次日涨停概率排序输出 Top50。当前采用规则评分模型，并在历史样本足够时执行历史命中率主导校准。

当前主流程：

1. 从 `a-share-top3-data/data/wp/latest/wp_latest_rank_input.csv` 读取候选输入。
2. 硬过滤：今日涨幅超过 8%、前一日未涨停、今日未涨停、成交额达到配置门槛，并排除 ST、停牌/无成交、退市整理、新股上市未满配置天数、严重数据缺失票。
3. 计算强板块、强个股、资金承接、动量、资金、形态、风险惩罚；字段包括成交额放大、日内收盘位置、尾盘/VWAP、3/5/10/20 日动量、均线位置、20 日突破、龙虎榜、热门题材等。
4. 输出 `p_limitup_t1`、`wp_score`、信号等级、核心理由和风险提示。
5. 生成 14:20 尾盘买入观察计划，最多 5 支，并给出确认条件和放弃条件。
6. 如存在足够历史回测样本，按概率区间命中率对 `p_limitup_t1` 做历史主导校准。
7. 生成 CSV、JSON、HTML，并由 GitHub Pages 发布。
8. 支持手动历史区间回测，输出 Top10/20/50、AUC、Brier、次日涨幅和回撤指标。

买入观察计划用于 14:20 手动运行后的人工决策流程。系统先按模型分、承接分、风险分、板块集中度和同板块数量约束筛出最多 5 支；用户在 14:20 到 14:50 继续观察，只有仍满足涨幅大于 8%、未涨停、尾盘不明显回落、成交承接不塌、同板块强势不退潮等条件时，才考虑执行。该模块不做自动下单。

实时模式会检查行情日期。如果上游 latest 行情日期不是当天交易日，报告显示“数据日期过期”，避免把旧行情误当成当天收盘数据。

自动任务会在 A 股交易时间窗口内运行，并内置 2026 年主要休市日过滤。若后续交易所调整休市安排，应同步更新日历配置。
