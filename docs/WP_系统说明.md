# WP 系统说明

WP 用于筛选今日涨幅超过 8%、前一日未涨停、今日未涨停的 A 股股票，并按尾盘次日收益条件排序输出 Top50。基础特征模型为 `wp_rule_v2_1`，尾盘排序模型为 `tail_profit_v1`。

当前主流程：

1. 从 `a-share-top3-data/data/wp/latest/wp_latest_rank_input.csv` 读取候选输入。
2. 硬过滤：今日涨幅超过 8%、前一日未涨停、今日未涨停、成交额达到配置门槛，并排除 ST、停牌/无成交、退市整理、新股上市未满配置天数、严重数据缺失票。
3. 计算强板块、强个股、资金承接、动量、资金、形态、风险惩罚；字段包括成交额放大、日内收盘位置、尾盘/VWAP、3/5/10/20 日动量、均线位置、20 日突破、龙虎榜、热门题材等。
4. 保留 `wp_score`、`ranking_score` 和 `p_limitup_t1` 作为解释字段。
5. 在全部候选中计算 `tail_profit_score`，硬限制涨幅不超过 12%、风险分不超过 45、收盘位置不低于 50、5 日成交额放大不超过 2.5 倍。
6. 按 `tail_profit_v1` 统一排序后截取 Top50，14:35 后最多生成 1 支主观察票；无合格票时空仓。
7. 生成 CSV、JSON、HTML，并由 GitHub Pages 发布。
8. 支持手动历史区间回测，输出 Top10/20/50、最终买入观察名单、AUC、Brier、次日涨幅和回撤指标。

尾盘观察计划用于 14:35 手动运行后的人工决策流程。用户观察到 14:50，只有涨幅仍大于 8%、未涨停、尾盘没有急回落且板块未转弱时才考虑执行。该模块不做自动下单。

历史日线代理回测只用于工程回归，不作为 14:35 实盘收益证明。累计验证只统计 `tail_profit_v1` 的最终尾盘名单，避免与旧模型记录混算。

实时模式会检查行情日期。如果上游 latest 行情日期不是当天交易日，报告显示“数据日期过期”，避免把旧行情误当成当天收盘数据。

自动任务会在 A 股交易时间窗口内运行，并内置 2026 年主要休市日过滤。若后续交易所调整休市安排，应同步更新日历配置。
